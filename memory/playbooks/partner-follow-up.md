# Partner Follow-Up

## Trigger

A partner, prospect, investor, collaborator, or warm contact needs follow-up.

## Required Inputs

- who the contact is
- relationship context
- last interaction
- desired outcome
- work or personal compartment

## Follow-Up Questions

Ask only if missing:

- target action wanted
- sensitivity level
- whether Rocky should draft only or request approval to send

## Routing Logic

- `Handshake Goblin` prepares relationship context and follow-up angle
- `Velvet Knife` drafts the message
- Rocky reviews and decides next step

## Approval Boundary

- draft: autonomous with receipts
- send: approval required every time

## Done Criteria

- follow-up draft exists
- relationship context is captured
- next action is clear
- send approval state is explicit

## Evidence Required

- source notes
- prior interaction summary
- rationale for timing and angle

## Output Format

- Task
- Assigned to
- Objective
- Inputs used
- Draft
- Result
- Evidence
- Risks or blockers
- Recommended next move
- Confidence level
