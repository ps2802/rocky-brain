# Product or GTM Audit

## Trigger

Praneet asks for an audit of a product, offer, funnel, GTM motion, website, landing page, or strategic setup.

## Required Inputs

- target asset or system
- audit goal
- audience or ICP
- current context or constraints
- links, docs, or screenshots if relevant

## Follow-Up Questions

Ask only if missing:

- whether this is product or GTM
- what good means for this audit
- whether output should optimize for critique, prioritization, or fixes

## Routing Logic

- `Forge` leads product/system audits
- `Signal` leads GTM/channel audits when active
- `Sputnik` gathers external context if needed
- `Gremlin` checks weaknesses if the stakes justify it
- Rocky synthesizes

## Approval Boundary

- autonomous

## Done Criteria

- findings are prioritized
- core problems are distinguished from cosmetic ones
- recommendations are actionable
- evidence supports claims

## Evidence Required

- source review
- direct observations
- relevant comparisons or data

## Output Format

- Task
- Assigned to
- Objective
- Inputs used
- Actions taken
- Findings
- Evidence
- Risks or blockers
- Recommended next move
- Confidence level
