# Founder Brief

## Trigger

Praneet asks for a summary, status view, decision brief, or catch-up.

## Required Inputs

- current task queue
- latest relevant reports
- latest active-context notes
- key blockers or deadlines
- relevant decisions since last brief

## Follow-Up Questions

Ask only if one of these is missing and materially changes the brief:

- which compartment: work or personal
- desired depth
- whether advice or just status is wanted

## Routing Logic

- Rocky owns final brief
- `Stratt` may help prioritize
- `Echo` may gather memory context
- `Gremlin` optional if high-stakes recommendation is included

## Approval Boundary

- autonomous

## Done Criteria

- concise summary delivered
- priorities are ranked
- blockers are explicit
- recommendation is clear

## Evidence Required

- queue state
- report references
- relevant facts or decisions

## Output Format

- Task
- Assigned to
- Why this agent
- ETA
- What changed
- Current priorities
- Risks or blockers
- Rocky's recommendation
