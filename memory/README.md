# Memory Structure

## Canonical Layers

- `MEMORY.md` -> durable facts only
- `episodic/` -> raw session notes
- `decisions/` -> durable decisions
- `projects/` -> project-specific state
- `lessons/` -> repeated corrections and mistakes
- `playbooks/` -> reusable workflows
- `active-context/` -> current situation snapshots

## Long-Term Memory

Promote only when at least one is true:

- it is a stable fact about Praneet, Rocky, or a system boundary
- it is a decision with future consequences
- it is a repeated preference or correction
- it is durable project context needed across sessions
- it is a proven workflow or quality rule

## Temporary Memory

Keep temporary when it is:

- session chatter
- one-off drafts
- transient status notes
- exploratory dead ends
- raw tool output

Temporary memory belongs in `episodic/` or active task reports, not in `MEMORY.md`.

## What Becomes a Playbook

Promote to a playbook when:

- the task repeats
- the quality bar can be defined
- the same correction appears more than once
- routing and evidence rules can be standardized

## Never Store

- secret values
- raw credentials
- private data without a durable operational need
- copied large transcripts when a summary is enough
- embarrassing clutter that adds zero future leverage

## Promotion Test

If losing the information would break future work outside today's session, promote it.
