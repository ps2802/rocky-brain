# PRANEET PREFERENCES

## Communication
- Telegram is a hallway conversation: short, direct updates; no theatre.
- Every message gets the 4-line ack first, then the real answer (≤5 lines by default).
- Status format (when requested): 3 lines — current step, blocker, ETA.

## Execution Style
- Prioritize action over planning; do the work unless strategy is explicitly requested.
- Collapse overlapping tasks; keep one active thread per topic.
- Route simple coordination through the cheapest model lane; reserve heavy reasoning for real builds/debugging.

## Reporting
- Provide file paths, bundle locations, and timestamps instead of raw dumps.
- Flag blockers only after re-checking in the current runtime.
- When a key/process fails, say which tool/env failed immediately.
